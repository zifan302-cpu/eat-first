# 冰箱小鲜队（Fresh Squad / Eat First）

一款本地优先、单人优先、轻游戏化的冰箱管理 PWA。它不追求记录一切，而是每天帮助用户回答一个问题：**冰箱里什么应该先处理？**

V0.8 是“灵活菜谱决策版”：系统先建议值得处理的食材，用户只在需要时调整必须使用、可以使用或本次排除；菜谱生成同时读取更完整的厨具档案、常备调料策略和本次做饭目标。

![冰箱小鲜队端正鲜物角色家族](assets/character-system/fresh-squad-family-style-reference.png)

## 当前产品能力

- 添加、编辑和删除冰箱食物
- 记录 `Use by`、`Best before`、开封日期或无日期食物
- 根据日期类型和时间进行本地优先级排序
- 每日首页展示最值得处理的 Top 3
- 支持使用一部分、全部用完、冷冻、丢弃和稍后处理
- 结构化记录数量和单位，并兼容旧版自由文本分量
- 高密度冰箱清单、名称搜索、日期与分类筛选
- 支持撤销最近一次关键操作
- 五种已验收案例食材的角色化快捷添加
- 今日处理任务、连续行动、小队等级和成长能量
- 小鲜队住处根据当前库存展示最多五种代表食材，同类合并，其余以数量汇总
- 番茄四阶段状态试点，并在列表和住处中根据优先级变化
- 摄像头扫描或手动输入 EAN / UPC / GS1 条形码，并从 Open Food Facts 生成可编辑录入草稿
- 根据建议、必须、可用和排除食材，以及菜系、人数、用时、饮食要求、做饭目标、厨具和常备调料，通过千问生成 2 个菜谱方案
- 所有当前食材都能在高级面板中调整角色，自动请求最多携带 15 项普通候选，不再固定为五项搭配
- 设置中可保存 16 种厨具和 9 类常备项，生成页默认只显示摘要，需要时才临时调整
- 固定列表之外还可以保存自定义厨具和自定义常备调料；它们只作为本地偏好，不会变成第二套库存
- 中文自动模式偏向中式家常，英文自动模式偏向全球日常，也可独立选择菜系
- 菜谱偏好持久化，弹窗内仍可只为本次临时调整
- 设置页分组折叠，支持文件式备份与恢复、安装状态识别和二次确认清空
- 添加食材草稿可在意外刷新后恢复
- 本周及时吃掉或冷冻的行动反馈
- 首次使用欢迎与产品价值引导
- 中英文界面
- 本地数据导入、导出和清空
- PWA 安装及离线基础支持

食品安全边界：本产品只做提醒和排序，不判断食物是否安全。用户始终应以包装标签和官方食品安全建议为准。

## 产品边界

现阶段聚焦单人、单设备、冰箱内食物的低负担管理。轻游戏化用于帮助养成处理食物的习惯，不用排行榜、惩罚性机制或复杂虚拟经济增加压力。

## 本地开发

```bash
npm install
npm run dev
```

在 Windows PowerShell 中使用 `npm.cmd run dev`（避免执行策略拦截 `npm.ps1`），它会同时启动 Vite 页面和同源 API。服务默认绑定本机 `127.0.0.1`；如果 `PORT` 被 Windows 保留或占用，会让系统分配可用端口并打印实际地址。需要让手机通过局域网访问时，可在 `.env.local` 设置 `HOST=0.0.0.0`。条形码查询开箱可用；摄像头在手机上需要 HTTPS 或 `localhost` 安全上下文。

### 配置千问

1. 复制 `.env.example` 为 `.env.local`。
2. 在 `.env.local` 中填写 `QWEN_API_KEY`，以及该密钥所属地域和工作空间对应的 OpenAI 兼容 `QWEN_BASE_URL`。
3. 如账户没有默认模型的权限，修改 `QWEN_MODEL`。
4. 重启 `npm run dev`。

`.env.local` 已加入 `.gitignore`。密钥只由 `server/index.mjs` 读取，不会进入 React、浏览器存储、请求日志或前端打包产物。请不要把真实密钥写入源码或提交到 Git。

常用验证命令：

```bash
npm test
npm run build
```

## 数据说明

食材清单和菜谱偏好默认保存在当前浏览器的 `localStorage` 中。条形码查询会把编码发送给 Open Food Facts；生成菜谱时，只会把用户选择的优先与辅助食材、现实约束和菜谱偏好，经自有服务端发送给已配置的千问接口。API 密钥不会发送给浏览器。旧 V1.0—V0.6 数据会继续迁移保留；小队等级、任务和连续行动均从真实行为记录计算，不需要后端账号。

## V0.5 游戏化原则

- 现实行动优先：只有吃掉、冷冻、丢弃和整理记录才能推动反馈。
- 正向而不惩罚：断开连续行动不会扣分，丢弃也不会降低等级。
- 角色服务信息：角色用于说明任务、提醒和结果，不遮挡核心食物信息。
- 住处承载情感：小队有独立生活场景，但成长只映射真实冰箱行动。
- 不做复杂经济：当前没有商城、抽卡、体力、排行榜或付费加速。
