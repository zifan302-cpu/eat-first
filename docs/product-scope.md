# 产品边界

## 产品定义

冰箱小鲜队是一款优先服务单人冰箱的轻游戏化管理工具。核心价值不是建立完整库存账本，而是用尽量少的录入成本，持续给出“今天优先处理什么”的清晰建议，并用温和反馈帮助用户减少食物浪费。多人共享冰箱属于后续扩展，不进入当前主线。

## 当前主线

1. 快速记录：让添加、更新和完成一项食物尽可能省力。
2. 今日决策：可靠地排出 Top 3，并解释为什么优先。
3. 行为闭环：记录吃掉、冷冻、丢弃、稍后等真实结果。
4. 轻游戏化：通过连续行动、每周小结、收藏进度等正向反馈培养习惯。
5. 单人、本地优先：当前无需账号即可使用，数据默认留在设备中。

## 暂不进入主线

- 食品安全判定、医疗或营养建议
- 完整菜谱平台、购物清单或家庭 ERP
- 为展示而存在的假付费墙、演示解锁码和样例冰箱
- 复杂积分商城、竞争排行榜或惩罚性连续签到
- 当前阶段的多人共享、后端、账号体系和多端同步

## V0.4 已落地

- 常见食物快捷模板和更清晰的连续添加
- 今日 Top 3 与“今日拯救行动”任务
- 以“端正鲜物”为标准的五种案例食材和统一生成母版
- 全页面编辑式纸张 UI，替换旧的黏土角色与玻璃渐变界面
- 小队等级、能量、连续行动和本周反馈
- 最近一次关键操作撤销
- 首次使用欢迎页与新版番茄 PWA 图标

## V0.5 已落地

- 独立的“小鲜队住处”页面，让五位角色拥有稳定的生活场景
- 住处只映射今日任务、小队等级和真实拯救记录，不加入虚构经济
- 成果页继续保留，并从住处进入；底部导航优先呈现核心管理与养成入口
- 角色源图移出 PWA 公共目录，安装包只保留经过处理的透明运行时资产
- 预留视觉录入窗口：未来由图片生成可编辑草稿，确认后才写入本地数据
- 预留 AI 菜谱窗口：未来根据选中食材、用时、人数和饮食限制提供少量建议

两项 AI 能力当前只开放流程预览，不调用模型、不上传图片，也不生成伪结果。模型选择、隐私策略、成本控制与实测准确率需在正式接入前单独评审。

## 后续候选方向

- A. 批量添加与可编辑的个人食物模板
- B. 本地通知、临期提醒和每日处理计划
- C. 更多角色状态、阶段性成就和轻量收藏
- D. 后续增强协作：共享冰箱与家庭成员操作记录

已确认优先顺序：继续完成单人冰箱的核心体验，再考虑多人功能。D 需要后端与账号体系，待单人版稳定后再评估。
