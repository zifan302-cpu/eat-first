# 冰箱小鲜队 V0.5 设计系统

## 设计方向：端正鲜物编辑部

V0.5 以已确认的“端正鲜物”角色参考图为视觉宪法。界面不再使用旧的光滑 2.5D 黏土角色、玻璃拟态和活动页式渐变，而是把产品设计成一本每天更新的冰箱行动小刊物，并给小队一个独立生活场景。

三个关键词：**端正、克制、可亲近**。

- 端正：信息和角色都使用稳定重心，用户一眼知道先做什么。
- 克制：角色只出现在有叙事价值的位置，游戏化不压过真实食物信息。
- 可亲近：蛋壳纸底、自然食材色和轻微手工纹理降低工具感，但不幼儿化。

## 界面语法

- 页面底色：暖蛋壳纸 `#F7EEDC`，叠加极轻微纸张颗粒。
- 卡片底色：柔和纸白 `#FCF8ED`。
- 主文字与主按钮：深墨绿 `#203D2E`。
- 分隔线：纸板灰棕 `#D8CBB5`。
- 主绿色：叶绿 `#526B45`，只用于稳定、正常和选中反馈。
- 食材强调：番茄 `#D94A2F`、胡萝卜 `#D9771F`、茄紫 `#57445F`、蘑菇棕 `#9A6555`。
- 卡片使用一像素边线和轻微右下偏移阴影，不使用玻璃模糊与大面积发光。
- 圆角保持柔和但不过度膨胀；主要卡片约 22px，按钮约 16px。
- 标题使用紧凑的编辑式字距，正文保持高可读性。

## 页面结构

- 首页：日期与品牌 → 今日行动 → 三项真实数据 → Eat First 列表。
- 添加：案例食材横向样架 → 单一录入表单。
- 冰箱：页面说明 → 日期标签筛选 → 当前食物卡。
- 小队：冰箱公寓静态场景 → 今日状态 → 成员、等级和真实拯救记录。
- 成果：连续行动说明 → 四项记录 → 小队能量 → 最近行动。
- 设置：本地守护角色 → 语言、安装、数据、安全和版本。

## 角色使用

- 正式参考：`assets/character-system/` 下两张已确认母版；该目录不进入 PWA 构建。
- 产品透明角色：`public/art/upright-characters/`。
- 首批案例食材：番茄、胡萝卜、西兰花、茄子、蘑菇。
- 未建立正式角色母版的食物使用文字首字占位，不借用错误食材角色冒充。
- 食材形象不穿衣、不拿职业道具，不依靠徽章制造身份。
- 角色循环动画只允许 1–2px 的缓慢呼吸，不弹跳、不挥手。

## 游戏化边界

V0.5 保留四种轻量机制：

1. 今日最多三个真实处理决定。
2. 吃掉或及时冷冻转化为小队成长记录。
3. 连续行动只记录节奏，断开不扣分，丢弃不惩罚。
4. 独立住处提供轻养成情感底色，当前保持静态，不伪造互动与奖励。

不加入商城、抽卡、体力值、排行榜、付费加速或焦虑式签到。

## 可访问性

- 所有状态同时提供文字，不只依赖颜色和角色表情。
- 主要点击目标不小于 44px。
- 动画遵守 `prefers-reduced-motion`。
- 深墨绿与纸白保持足够正文对比度。
